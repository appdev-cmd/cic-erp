1. Kiến trúc phân quyền 5 tầng

1
Xác thực (Authentication)
Google OAuth → Email @cic.com.vn → Employee lookup
2
Vai trò (Roles)
profiles.role → 9 vai trò (Admin, Leadership, UnitLeader, AdminUnit, NVKD, NVKT...)
3
Phân quyền chi tiết (RBAC)
user_permissions → resource × action (view / create / update / delete)
4
Phạm vi dữ liệu (Data Scoping)
profiles.unit_id + cross_unit_visibility → giới hạn dữ liệu theo đơn vị
5
Kiểm toán (Audit Log)
audit_logs → ghi nhận mọi thay đổi với old_data / new_data

2. Xác thực (Authentication)

Luồng đăng nhập

1
Google OAuth Login qua Supabase Auth
2
Kiểm tra email domain → phải là @cic.com.vn
3
Tìm hồ sơ trong bảng profiles (theo auth user ID)
4
Tìm nhân viên trong bảng employees (theo email)
5
Tự động liên kết profile ↔ employee, đồng bộ role, unit_id
6
Fetch unit code từ bảng units → Set UserProfile
Lưu ý: Người ngoài công ty hoặc nhân sự chưa được tạo hồ sơ sẽ không thể đăng nhập.

3. Vai trò người dùng (9 vai trò)

🔑
Quản trị hệ thống Admin
🌐 Toàn công ty
Toàn quyền: cài đặt, phân quyền, quản lý user, CRUD tất cả. Bypass mọi permission check.
👔
Ban lãnh đạo Leadership
🌐 Toàn công ty
Xem báo cáo tổng hợp, quản lý toàn công ty, CRUD dữ liệu, phê duyệt HĐ.
🏢
Lãnh đạo đơn vị UnitLeader
🏢 Đơn vị mình
Giám đốc/Phó GĐ đơn vị, sửa tất cả HĐ trong đơn vị, phê duyệt PAKD.
📋
Admin đơn vị AdminUnit
🏢 Đơn vị mình
Hỗ trợ vận hành, nhập liệu cho đơn vị. Nếu thuộc HCNS → truy cập module Nhân sự.
💼
NV Kinh doanh NVKD
🏢 Đơn vị mình
Tạo/theo dõi hợp đồng, khách hàng. Chỉ sửa HĐ do chính mình tạo/được phân công.
🔧
NV Kỹ thuật NVKT
🏢 Đơn vị mình
Triển khai kỹ thuật, hỗ trợ thực hiện HĐ. Quản lý SP/DV. Không tạo/sửa HĐ, không nhập thanh toán.
💰
Kế toán trưởng ChiefAccountant
🌐 Toàn công ty
Phê duyệt tài chính, toàn quyền thanh toán thực tế, xóa thanh toán, xem nhân sự.
📊
Kế toán Accountant
🌐 Toàn công ty
Ghi nhận thu chi thực tế, xuất hóa đơn doanh thu. Không xóa thanh toán.
⚖️
Pháp chế Legal
🌐 Toàn công ty
Rà soát pháp lý hợp đồng (chỉ xem). VCU customers/products.

5. Phạm vi dữ liệu (Data Scoping)

🌐 Nhóm Global View

Admin, Leadership, Legal, Accountant, ChiefAccountant

  • Xem dữ liệu tất cả đơn vị
  • getVisibleUnitIds() = null

🏢 Nhóm Unit-Scoped

NVKD, NVKT, AdminUnit, UnitLeader

  • Chỉ xem dữ liệu đơn vị mình
  • + đơn vị được cấp quyền cross-unit
  • getVisibleUnitIds() = [ownUnit, ...]

Quyền xem liên đơn vị (Cross-Unit Visibility)

Cross-unit chỉ cho phép xem (view), không bao gồm quyền tạo/sửa/xóa dữ liệu của đơn vị khác. Chỉ Admin quản lý cross-unit visibility.

Quy tắc đặc biệt cho Hợp đồng

1
Kiểm tra DB: can('contracts', 'update') → false? → TỪ CHỐI
2
Admin hoặc Leadership? → CHO PHÉP (tất cả HĐ)
3
Accountant / ChiefAccountant? → CHO PHÉP (chỉ trường tài chính)
4
Legal hoặc NVKT? → TỪ CHỐI (chỉ xem)
5
HĐ khác đơn vị? → TỪ CHỐI
6
AdminUnit / UnitLeader? → CHO PHÉP (tất cả HĐ đơn vị)
7
NVKD? → CHO PHÉP nếu salespersonId === employeeId

7. Quy trình quản trị

Thêm nhân viên mới

1
Admin tạo hồ sơ nhân sự (employees) với email @cic.com.vn
2
Nhân viên đăng nhập Google OAuth lần đầu
3
Hệ thống tự động: tạo profile, liên kết employee, gán role + unit_id
4
DB seed default permissions theo role
5
Nhân viên sử dụng hệ thống với quyền mặc định ✅

Thay đổi quyền cá nhân

1
Admin → Cài đặt → Phân quyền người dùng
2
Chọn user → Bật/tắt từng action trên từng resource
3
Upsert vào user_permissions → Audit log ghi nhận

Cấp quyền xem liên đơn vị

1
Admin → Cài đặt → Quyền xem liên đơn vị
2
Chọn nhân viên → Tick các đơn vị được phép xem
3
Nhân viên thấy dữ liệu đơn vị bổ sung (chỉ xem, không sửa)

8. Phụ lục

Ký hiệu viết tắt

Ký hiệu Ý nghĩa
V View (xem)
C Create (tạo)
U Update (sửa)
D Delete (xóa)
VCU View + Create + Update
VCUD View + Create + Update + Delete
HĐ Hợp đồng
PAKD Phương án kinh doanh
HCNS Hành chính Nhân sự (Phòng Tổng hợp)
TCKT Tài chính Kế toán

Bảng cơ sở dữ liệu liên quan

Bảng Mô tả
profiles Thông tin user (role, unit_id, employee_id)
user_permissions Quyền chi tiết (resource × actions)
cross_unit_visibility Quyền xem liên đơn vị
employees Hồ sơ nhân sự
units Đơn vị / phòng ban
audit_logs Nhật ký hoạt động
