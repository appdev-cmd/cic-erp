1. Tổng Quan Công Nghệ

⚛️ React 19

Thư viện xây dựng giao diện. Chia giao diện thành các component nhỏ, tái sử dụng được.

Ví dụ minh họa
Trang Danh sách hợp đồng = 1 component ContractList.tsx
Nút "Thêm mới" bên trong = 1 component Button nhỏ hơn

🟦 TypeScript

JavaScript nâng cấp, kiểm tra kiểu dữ liệu. File .ts = logic, .tsx = giao diện.

Ví dụ
types.ts định nghĩa: Hợp đồng có title (text), value (số), status (trạng thái)

⚡ Vite

Công cụ chạy dự án. Gõ npm run dev → mở trình duyệt xem ứng dụng.

Các lệnh
npm run dev = chạy thử  |  npm run build = đóng gói

🟩 Supabase

Backend-as-a-Service: cơ sở dữ liệu (PostgreSQL), đăng nhập, lưu file, realtime — không cần tự xây server.

Ví dụ
Khi bấm "Lưu hợp đồng" → gọi API Supabase → lưu vào bảng contracts trong database

🎨 TailwindCSS

Tạo giao diện bằng class có sẵn, không cần viết file CSS riêng.

Ví dụ class
bg-white = nền trắng  |  dark:bg-slate-900 = nền tối trong dark mode
text-lg = chữ lớn  |  rounded-lg = bo góc

🚀 Vercel

Nền tảng deploy ứng dụng lên internet. Push code → tự động build và cập nhật website.

Quy trình
Code → Push GitHub → Vercel tự build → Website cập nhật (2-3 phút)

2. Frontend — Giao Diện Người Dùng

Frontend = tất cả những gì bạn nhìn thấy và tương tác trên trình duyệt.

Các thành phần giao diện cơ bản

📦 Component

Khối giao diện độc lập, tái sử dụng. Mỗi file .tsx = 1 component.

Cách nói
"Sửa component danh sách hợp đồng"

📄 Page (Trang)

Trang đầy đủ khi truy cập 1 URL.

Cách nói
"Mở trang Dashboard" "Trang chi tiết hợp đồng"

📝 Form (Biểu mẫu)

Giao diện nhập liệu: ô input, dropdown, nút Lưu.

Cách nói
"Form tạo hợp đồng mới" "Sửa form thanh toán"

💬 Modal / Dialog

Cửa sổ popup bật lên để xác nhận hoặc nhập nhanh.

Cách nói
"Modal xác nhận xóa" "Dialog thêm KH nhanh"

📋 List (Danh sách)

Bảng hiển thị nhiều bản ghi dạng hàng/cột.

Cách nói
"Danh sách thanh toán" "Bảng hợp đồng"

🔍 Detail (Chi tiết)

Trang xem đầy đủ thông tin 1 bản ghi.

Cách nói
"Trang chi tiết khách hàng"

📑 Tab

Các thẻ chuyển đổi nội dung trong cùng 1 trang.

Cách nói
"Tab Phương án KD" "Chuyển sang tab Thanh toán"

📌 Sidebar

Thanh menu bên trái để chuyển giữa các module.

Cách nói
"Thêm mục Báo cáo vào sidebar"

🔝 Header

Thanh trên cùng: logo, tên user, chọn đơn vị.

Cách nói
"Thêm nút thông báo vào header"

4. Database — Cơ Sở Dữ Liệu

Thuật ngữ cơ sở dữ liệu

📊 Table (Bảng)

Tập hợp dữ liệu cùng loại: bảng contracts, customers...

Tưởng tượng
Như 1 sheet Excel — mỗi sheet = 1 table

📏 Column (Cột)

Trường dữ liệu: name, status, value...

Tưởng tượng
Như tiêu đề cột trong Excel: Tên, Trạng thái, Giá trị

📝 Row (Dòng)

Một bản ghi cụ thể (1 hợp đồng, 1 khách hàng).

Tưởng tượng
Mỗi dòng trong Excel = 1 bản ghi

🔗 Foreign Key (Khóa ngoại)

Liên kết 2 bảng: customerId trong bảng HĐ trỏ đến bảng KH.

Ví dụ
HĐ có customerId = "abc" → tìm KH có id = "abc"

6. UX — Trải Nghiệm Người Dùng

UX = ứng dụng có dễ dùng hay không, phản hồi có nhanh hay không.

⏳ Loading State

Trạng thái đang tải — hiển thị skeleton hoặc spinner.

Cách nói
"Trang bị loading mãi không xong" hoặc "Thêm skeleton khi tải"

❌ Error State

Khi có lỗi xảy ra — hiển thị thông báo lỗi.

Cách nói
"Khi lỗi mạng, hiện trang trắng, cần hiển thị thông báo lỗi"

📭 Empty State

Khi không có dữ liệu nào.

Cách nói
"Khi chưa có HĐ nào, hiện trống, cần thêm hình minh họa + nút Tạo mới"

🔔 Feedback (Phản hồi)

Phản hồi sau khi user thao tác: toast "Đã lưu", "Đã xóa".

Cách nói
"Bấm Lưu xong không thấy thông báo gì"

🔍 Filter & Search

Lọc + tìm kiếm dữ liệu.

Cách nói
"Thêm bộ lọc theo trạng thái" hoặc "Ô tìm kiếm không ra kết quả"

📊 Pagination & Sort

Phân trang + sắp xếp cột.

Cách nói
"Phân trang 20 bản ghi/trang" hoặc "Sắp xếp theo giá trị giảm dần"

10. Mẹo Mô Tả Yêu Cầu Cho AI

🎯 Công thức vàng
[Hành động] + [Đối tượng] + [Vị trí] + [Chi tiết]
• "Thêm [nút Xuất PDF] vào [trang chi tiết HĐ] [góc trên bên phải, cạnh nút Sửa]"
• "Sửa [dark mode] của [bảng danh sách thanh toán] [text không đọc được trên nền tối]"
• "Thêm [cột ngày thanh toán] vào [bảng payments] [kiểu date, sau cột số tiền]"

✅ Mô tả TỐT

  • "Sửa component PaymentList: thêm cột Đơn vị"
  • "Bug form tạo HĐ: dropdown KH không hiển thị"
  • "Thêm feature xuất Excel cho danh sách HĐ"
  • "Dark mode bị lỗi ở bảng ContractList"
  • "Role Accountant cần quyền delete payments"
  • "Thêm cột note kiểu text vào bảng payments"

❌ Mô tả KHÔNG TỐT

  • "Sửa trang thanh toán" → Sửa gì? UI? Logic? Data?
  • "Thêm cái nút" → Nút gì? Ở đâu? Làm gì khi bấm?
  • "Không chạy được" → Lỗi gì? Ở đâu? Thông báo?
  • "Sửa cho đẹp hơn" → Đẹp kiểu nào? Phần nào?
  • "Bị sai" → Cái gì sai? Mong đợi gì? Thực tế gì?
  • "Update lại" → Update cái gì? Ở đâu?

💡 Mẹo cuối cùng

Khi không biết thuật ngữ chính xác:
• Chụp màn hình và gửi kèm mô tả
• Mô tả vị trí: "cái ô vuông góc trên bên trái", "cái bảng ở giữa trang"
• Mô tả hành vi: "khi bấm vào thì không có gì xảy ra"
→ AI sẽ hiểu! 🎯
